## 4. 💬 溝通與工具規範 (Communication & Tools)
*定義什麼資訊該出現在哪裡，避免訊息淹沒。*

| 資訊類型 | 建議管道 | 規範 |
| :--- | :--- | :--- |
| **碎碎念/靈感** | Line / Discord | 雜訊高，主要用於即時討論與快速確認。 |
| **緊急事項** | 電話或語音 | 直接切入重點。 |
| **正式結論** | **Markdown 檔案** | **預設用檔案溝通**，所有共識與結論必須定錨於 Repo (SSOT)。 |
| **任務追蹤** | `TASKS.md` | 所有待辦與認領點的正式紀錄。 |
| **異步非同步討論** | 論壇 / 議題單 (Issues) | 適合長篇、具備脈絡且需要留存的討論。 |

### 🛠 團隊標準資訊工具清單 (Standard Info Tools)
*   **版本控制與協作**：Git (GitHub/GitLab)，確保產出可回溯。
*   **檔案編修**：Visual Studio Code / Obsidian (Markdown 語法)。
*   **即時通訊**：Discord (主要討論頻道)、Line (緊急聯繫)。
*   **AI 助手**：Antigravity (主動式任務管理)、Claude/ChatGPT (輔助產出)。
*   **視訊會議**：Google Meet / Zoom。
*   **雲端存儲**：Google Drive (放置大型非文字素材)。

## 5. 📂 檔案結構與 Git 規範 (Structure & Git)
*確保團隊成員不會找不到檔案，且 AI 不會改爛東西。*

*   **主要資料夾結構**：
    *   `/workmgr/tasks/`：存放任務清單。
    *   `/docs/`：存檔正式檔案、草稿。
    *   `/src/`：存放程式碼與腳本。
*   **Git 協作原則**：
    *   **Layer 0 底座**：每次大改前必須 commit。
    *   **Commit 格式**：`[任務ID] 簡短描述` (方便 AI 追蹤)。

## 7. 🤖 AI 協作角色 (AI Role)
*定義團隊中的 AI 該做什麼，不該做什麼。*

*   **AI 負責**：自動撰寫會議摘錄、檢查 Git 衝突、提醒過期任務。
*   **AI 不得**：未經人類確認直接將大改動 Push 到 `main` 分支。
